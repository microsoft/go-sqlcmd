### PR #684 - fix: reject non-YAML extensions for --sqlconfig flag
**Status: PASS - minor notes, no must-fix**

Reviewed: 2026-04-24

Changes: Validates file extension in `config.SetFileName()` - rejects non-.yaml/.yml extensions. Updates `--sqlconfig` help text. Adds README section. Adds test. Regenerates translations.

Findings:
- **Validation logic is correct.** Extension check uses `strings.ToLower(filepath.Ext(name))`, handles case-insensitive extensions. Extensionless files (like default `sqlconfig`) pass through.
- **Placement is correct.** Validation happens before `filename = name` and `file.CreateEmptyIfNotExists()`, so bad-extension files are never created.
- **Error path is correct.** `checkErr(fmt.Errorf(...))` uses the error callback which panics in init/test context, fatals in production. `return` after `checkErr` is defensive (unreachable if callback panics/exits, safe otherwise).
- **MINOR: Test leaves files.** `assert.NotPanics` creates `~/.sqlcmd/sqlconfig-test-yaml.yaml` and `sqlconfig-test-yml.yml` that are never cleaned up. Pre-existing pattern - all config tests create files in `~/.sqlcmd/` via `SetFileNameForTest`. Not a regression.
- **MINOR: Translations blanked.** Non-English translations for "configuration file" are replaced with empty translations for the new longer string. Expected gotext behavior - needs translator follow-up.
- **MINOR: Import reordering.** stdlib imports grouped above third-party in config.go and config_test.go. Correct goimports style but gratuitous churn.
